# 2026-03-11 — Agent Architecture Redesign

## Core Shift: Workflows as Policy Groups
Moved from role-centric agent templates to **workflow-centric policy groups**:
- Workflows define capability sets (the "Rule of Two": no agent gets `file` + `search` + `code`)
- Agents are semi-permanent entities assigned to workflow rosters
- At spawn: task → decomposition → workflow selection → agent from roster → provisioned with workflow capabilities

## Capability Profiles (Rule of Two)
| Set | Capabilities | Use Case |
|-----|--------------|----------|
| `read-search` | `file` + `search` | Research, information gathering |
| `read-execute` | `file` + `code` | Implementation, local changes |
| `read-only` | `file` | Audit, review, measurement |

## Workflows Defined
- `research` → `read-search` capabilities
- `implement` → `read-execute` capabilities  
- `audit` → `read-only` capabilities
- `integrate` → two-phase: research (`read-search`) → handoff → implement (`read-execute`)
- `orchestrate` → `read-only` (coordinator spawns others for real work)

## Handoff Protocol
Agent completes phase → returns artifact + context + recommended next phase → orchestrator evaluates → spawns next agent with appropriate capabilities. **No mid-flight capability changes.**

## Meta
